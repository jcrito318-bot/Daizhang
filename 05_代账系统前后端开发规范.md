# 代账系统前后端开发规范文档

## 一、后端开发规范

### 1.1 Java开发规范

#### 1.1.1 代码风格规范

**命名规范**
- 类名：使用大驼峰命名法（PascalCase）
  - 实体类：`AccountSet`、`Voucher`、`Subject`
  - Controller类：`VoucherController`
  - Service类：`VoucherService`
  - Service实现类：`VoucherServiceImpl`
  - Mapper类：`VoucherMapper`
  - DTO类：`VoucherCreateRequest`、`VoucherUpdateRequest`
  - VO类：`VoucherVO`、`AccountSetVO`

- 方法名：使用小驼峰命名法（camelCase）
  - 查询单个：`getVoucher()`
  - 查询列表：`listVouchers()`
  - 查询分页：`pageVouchers()`
  - 新增：`createVoucher()`
  - 修改：`updateVoucher()`
  - 删除：`deleteVoucher()`
  - 判断：`isVoucherValid()`、`hasPermission()`

- 变量名：使用小驼峰命名法
  - 普通变量：`accountSetId`、`voucherDate`
  - 集合变量：`voucherList`、`subjectMap`
  - 布尔变量：`isActive`、`hasPermission`
  - 常量：`MAX_RETRY_COUNT`、`DEFAULT_PAGE_SIZE`

**代码格式**
- 缩进：使用4个空格，禁止使用Tab
- 行宽：每行代码不超过120个字符
- 空行：方法之间空一行，逻辑代码块之间空一行
- 大括号：左大括号前不换行，右大括号单独一行

```java
// 正确示例
public class VoucherController {
    
    @Autowired
    private VoucherService voucherService;
    
    @GetMapping("/{id}")
    public Result<VoucherVO> getVoucher(@PathVariable Long id) {
        VoucherVO voucher = voucherService.getVoucher(id);
        return Result.success(voucher);
    }
}
```

#### 1.1.2 分层架构规范

**Controller层规范**
- 只负责接收请求参数、调用Service、返回响应结果
- 禁止在Controller层编写业务逻辑
- 使用`@Valid`注解进行参数校验
- 统一使用`Result`包装返回结果

```java
@RestController
@RequestMapping("/api/v1/vouchers")
@Tag(name = "凭证管理")
public class VoucherController {
    
    @Autowired
    private VoucherService voucherService;
    
    @PostMapping
    @Operation(summary = "创建凭证")
    public Result<VoucherVO> createVoucher(
            @RequestBody @Valid VoucherCreateRequest request) {
        VoucherVO voucher = voucherService.createVoucher(request);
        return Result.success(voucher);
    }
    
    @GetMapping
    @Operation(summary = "分页查询凭证")
    public Result<PageResult<VoucherVO>> pageVouchers(
            @Valid VoucherQueryRequest request) {
        PageResult<VoucherVO> page = voucherService.pageVouchers(request);
        return Result.success(page);
    }
}
```

**Service层规范**
- 负责业务逻辑处理
- 事务控制放在Service层
- 禁止在Service层直接操作HttpServletRequest/Response
- 复杂业务逻辑需要添加详细注释

```java
@Service
@Transactional(rollbackFor = Exception.class)
public class VoucherServiceImpl implements VoucherService {
    
    @Autowired
    private VoucherMapper voucherMapper;
    
    @Autowired
    private VoucherDetailMapper voucherDetailMapper;
    
    @Override
    public VoucherVO createVoucher(VoucherCreateRequest request) {
        // 1. 校验借贷平衡
        validateBalance(request);
        
        // 2. 生成凭证号
        String voucherNo = generateVoucherNo(request.getAccountSetId(), 
                                              request.getYear(), 
                                              request.getMonth());
        
        // 3. 保存凭证主表
        Voucher voucher = new Voucher();
        BeanUtils.copyProperties(request, voucher);
        voucher.setVoucherNo(voucherNo);
        voucher.setStatus(VoucherStatus.UNAUDITED);
        voucherMapper.insert(voucher);
        
        // 4. 保存凭证明细
        saveVoucherDetails(voucher.getId(), request.getDetails());
        
        // 5. 返回结果
        return convertToVO(voucher);
    }
    
    private void validateBalance(VoucherCreateRequest request) {
        BigDecimal totalDebit = request.getDetails().stream()
                .map(VoucherDetailRequest::getDebit)
                .reduce(BigDecimal.ZERO, BigDecimal::add);
        
        BigDecimal totalCredit = request.getDetails().stream()
                .map(VoucherDetailRequest::getCredit)
                .reduce(BigDecimal.ZERO, BigDecimal::add);
        
        if (totalDebit.compareTo(totalCredit) != 0) {
            throw new BusinessException(3001, "凭证借贷不平衡");
        }
    }
}
```

**全局异常处理器**
```java
@Slf4j
@RestControllerAdvice
public class GlobalExceptionHandler {
    
    /**
     * 业务异常处理
     */
    @ExceptionHandler(BusinessException.class)
    public Result<Void> handleBusinessException(BusinessException e) {
        log.warn("业务异常：code={}, message={}", e.getCode(), e.getMessage());
        return Result.error(e.getCode(), e.getMessage());
    }
    
    /**
     * 参数校验异常处理
     */
    @ExceptionHandler(MethodArgumentNotValidException.class)
    public Result<Void> handleValidationException(MethodArgumentNotValidException e) {
        BindingResult bindingResult = e.getBindingResult();
        List<String> errors = bindingResult.getFieldErrors().stream()
                .map(error -> error.getField() + ": " + error.getDefaultMessage())
                .collect(Collectors.toList());
        
        log.warn("参数校验失败：{}", errors);
        return Result.error(400, "参数校验失败", errors);
    }
    
    /**
     * 系统异常处理
     */
    @ExceptionHandler(Exception.class)
    public Result<Void> handleException(Exception e) {
        log.error("系统异常：", e);
        return Result.error(500, "系统繁忙，请稍后重试");
    }
}
```

```java
    }

@Data

    private List<T> list;
    
    private Long total;
    
    private Integer pageNum;
```

### 1.2 数据库开发规范

#### 1.2.1 SQL编写规范

**基本规范**
- SQL关键字全部大写
- 表名、字段名使用小写
- 使用预编译语句，禁止SQL拼接
- 避免使用SELECT *，明确指定字段
- 复杂查询添加注释说明

```java
// 正确示例
@Select("SELECT id, account_set_id, code, name " +
        "FROM sub_subject " +
        "WHERE account_set_id = #{accountSetId} " +

List<Subject> selectSubjectList(@Param("accountSetId") Long accountSetId);

// 错误示例
@Select("select * from sub_subject where account_set_id = " + accountSetId)
```

**分页查询规范**
- 使用MyBatis Plus的分页插件
- 避免深度分页（LIMIT 1000000, 10）
- 大数据量分页使用游标分页

```java
// 正确示例
Page<VoucherVO> page = new Page<>(pageNum, pageSize);
Page<VoucherVO> result = voucherMapper.selectVoucherPage(page, query);

// 错误示例（深度分页）
SELECT * FROM vou_voucher LIMIT 1000000, 10
```

**批量操作规范**
- 批量插入使用`saveBatch`方法
- 批量更新使用`updateBatchById`方法
- 批量删除使用IN语句

```java
// 正确示例
List<Voucher> voucherList = new ArrayList<>();
// ... 构建voucherList
voucherService.saveBatch(voucherList, 100);

// 错误示例（循环单条插入）
```

### 1.3 接口开发规范

#### 1.3.1 RESTful API规范

**URL设计规范**
- 使用名词复数形式：`/api/v1/vouchers`
- 使用连字符分隔：`/api/v1/account-sets`
- 层级关系清晰：`/api/v1/vouchers/{id}/details`
- 避免使用动词：不使用`/api/v1/getVoucher`

**HTTP方法规范**
- GET：查询资源
- POST：创建资源
- PUT：更新资源（全量）
- PATCH：更新资源（部分）
- DELETE：删除资源

**状态码规范**
- 200 OK：请求成功
- 201 Created：创建成功
- 204 No Content：删除成功
- 400 Bad Request：请求参数错误
- 401 Unauthorized：未认证
- 403 Forbidden：无权限
- 404 Not Found：资源不存在
- 500 Internal Server Error：服务器错误

```typescript
  (config) => {
    const userStore = useUserStore()
    if (userStore.token) {
      config.headers.Authorization = `Bearer ${userStore.token}`
    }
    return config
  },
  (error) => {
    return Promise.reject(error)
  }
)

// 响应拦截器
service.interceptors.response.use(
  (response: AxiosResponse) => {
    const { code, message, data } = response.data
    
    // 业务成功
    if (code === 200) {
      return response.data
    }
    
    // 业务失败
    ElMessage.error(message || '请求失败')
    return Promise.reject(new Error(message || '请求失败'))
  },
  (error) => {
    // HTTP错误
    if (error.response) {
      const { status } = error.response
      
      switch (status) {
        case 401:
          ElMessage.error('未认证，请重新登录')
          router.push('/login')
          break
        case 403:
          ElMessage.error('无权限访问')
          break
        case 404:
          ElMessage.error('请求的资源不存在')
          break
        case 500:
          ElMessage.error('服务器错误')
          break
        default:
          ElMessage.error(`请求失败：${status}`)
      }
    } else {
      ElMessage.error('网络错误，请检查网络连接')
    }
    
    return Promise.reject(error)
  }
)
```

- refactor：代码重构
- test：测试相关
- chore：构建过程或辅助工具变动

**示例**
```bash
feat(voucher): 添加凭证创建功能

- 实现凭证创建接口
- 实现凭证借贷平衡校验
- 添加凭证创建页面

Closes #123
```
